## 为什么值得玩

- **熟悉又有变化**：保留斗地主的单张、对子、顺子、炸弹、火箭等压牌直觉，但每手牌都要兼顾得分与破境节奏。
- **每局都能构筑**：奇牌、心法、机关与锦囊会改变出牌策略，而不是只堆数值。
- **有长期目标**：八境正传、无尽论道、角色解锁、图鉴、成就、每日挑战与周常禁宗共同构成长期成长。
- **可分享流派**：构筑可导入 / 导出，方便复刻有趣的卡组思路。
- **开箱即玩**：静态站点架构，可直接部署到 GitHub Pages。

## 30 秒上手

1. 点击一张或多张手牌组成合法牌型；再次点击可轮换候选组合，`Shift + 点击` 可微调。
2. 用同类型、更大的牌压过守关者；炸弹和火箭可以越级压制。
3. 每次出牌按 **牌理 × 气 × 宗势** 结算分数，达到本关门槛即可破境。
4. 通关后挑选奇牌并在藏经阁补强心法，让下一关的出牌规则产生新的组合。

新玩家建议从「常道」开始，先完成前 2～3 境，再尝试围绕同花、连对、炸弹或机关构筑流派。

## 核心内容

| 模块 | 内容 |
| --- | --- |
| 八境正传 | 每境包含明局、暗局、宗主局三段挑战 |
| 无尽论道 | 正传后可挑战无限叠加的词条与门槛 |
| 奇牌构筑 | 多种稀有度奇牌、心法、机关、锦囊的组合选择 |
| 角色与流派 | 多位牌客与五宗主题，提供不同开局方向 |
| 长线成长 | 阅历、图鉴、成就、每日挑战、周常禁宗、构筑分享码 |
| 难度曲线 | 常道、险道、宗师、传说四档挑战 |

## 本地运行

项目没有构建步骤，任选一种静态服务器启动方式：

```bash
# Python 3
python -m http.server 8765

# 或 Node.js
npx serve .
```

然后打开 <http://127.0.0.1:8765/>。

> 也可以直接打开 `index.html`，但建议使用静态服务器，便于模拟线上资源加载行为。

## 项目结构

```text
paizhong/
├─ index.html          # 页面骨架与各游戏界面
├─ css/
│  ├─ game.css         # 主样式与桌面战斗布局
│  ├─ mobile.css       # 移动端 / 低高度窗口适配
│  └─ clean.css、feel.css、polish.css、shell.css
├─ js/
│  ├─ data.js          # 角色、关卡、奇牌、成就等静态数据
│  ├─ engine.js        # 发牌、牌型识别、计分与 AI 决策
│  ├─ game.js          # 对局流程、成长与存档
│  ├─ ui.js            # 渲染、交互与动画
│  └─ assets.js        # 素材映射与按页面预加载
└─ assets/             # 牌面、角色、界面、特效等美术资源
```

## 存档与数据

- 游戏不需要账号或后端服务。
- 进度存于当前浏览器的 `localStorage`（键名：`paizong_meta_v2`）。
- 清除浏览器站点数据会清除本地进度；可在游戏内使用导入 / 导出功能备份构筑。

## 部署到 GitHub Pages

1. Fork 本仓库，或创建一个空仓库并推送代码。
2. 在仓库中打开 **Settings → Pages**。
3. 选择 **Deploy from a branch**，分支选择 `main`，目录选择 `/ (root)`。
4. 保存后等待部署完成，访问：

```text
https://<你的 GitHub 用户名>.github.io/<仓库名>/
```

更新后执行：

```bash
git add .
git commit -m "your change"
git push
```

GitHub Pages 会自动重新发布。若浏览器仍显示旧样式，可使用 `Ctrl + F5` 强制刷新。

## 参与贡献

欢迎通过 Issue 与 Pull Request 一起把《牌宗》打磨得更好：

1. 先在 Issue 说明你发现的问题或准备实现的方向。
2. Fork 后创建主题明确的分支，例如 `fix/battle-hand-layout`。
3. 保持改动聚焦；涉及 UI 时请附带 1280×720 或手机断点的截图。
4. 提交前至少检查：能从首页进入一局战斗、可正常出牌、刷新后存档仍可读取。
5. 在 PR 中说明改动内容、验证方式和可能影响的界面。

特别欢迎：牌型与 AI 优化、关卡与奇牌设计、低分辨率适配、性能诊断、无障碍改进、文案与美术建议。

## 开发提示

- 游戏规则与数据优先修改 `js/data.js`、`js/engine.js`、`js/game.js`；界面问题集中在 `js/ui.js` 与 `css/`。
- 素材请放入 `assets/` 并通过 `js/assets.js` 注册，以便按页面预加载。
- 修改战斗布局时请优先验证 **1280×720、1366×768、1440×720**，并复测移动端断点。
